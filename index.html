Research assistant, most of the work was done independently with checkins and meetings. The literature was based on this Kimberley Bush medicine book by King & Horsfall (2023) to map traditional uses against phytochemical data in a review table. The book contains 326 plants, so covering that literature takes a substantial amount of time, with just two days per week over 12 weeks it was not expected that I would cover the whole book.

This work was sent to Liu and she was quite pleased with that result, however there was stereochemistry missing in the SMILES strings which required some additional cleaning up, which is documented in the deliverables.

Artefact 2: Python Script for SMILES Stereochemistry Fix

Miamiao Liu upon receiving the lotus_compounds_filtered dataset from before noticed an issue with the stereochemistry being missing from the SMILES codes.

That means there isn't enough detail in the structures, there are details being left out which matter, in order to update these SMILES codes I used a script.

The script went to a chemistry database (PubChem) and used the keys which were available for each compound to search for the accurate SMILES codes where applicable with a 98.5% success rate. The remaining 1.5% are flagged for manual review by not having a SMILES code.

A fallback was included if the InchiKey used on PubChem did not result in finding a SMILES code through attempting to use the title instead to improve reliability. Overall the script did better than expected and the issue was resolved quickly.

Artefact 3: Machine learning for molecular docking: Preliminary testing results

I am working on a research proposal for a machine learning topic, so I started conducting background research.

After conducting a little bit of research, I decided to test out a few tools, starting off with a basic test with static protein machine learning blind docking + physics based scoring.

Translation: I grabbed the old technology for how researchers predict if a drug binds to a protein (useful for finding drug targets which are needed to put drugs on the market). I tried out this old technology which uses the trial and error method for sticking drugs onto proteins, and you don't even need to tell the machine where to stick, it will do it blind (no hints).

This technology wasn't modern like the new co-folding method, which is like alphafold but with an extra step, we fold the protein and also the drug as well to predict drug targets. As a result, while the first test worked fine and I gathered some scores from my input data and validated the pipeline, it was time to try out this fun new technology.

This requires a lot of computational power, so I cracked open the piggy bank and invested into $1 worth of cloud compute in order to test it out for 1 hour using run-pod, which is a GPU-hosting service.

Boltz-2 was used on 8 real drugs and proteins yielding scores for 7 (1 failed due to bad input data).
